# Etapa 7 — Dashboard

## Checklist de execução

- [ ] Validar com a área de negócio a definição de cada card, percentual e status.
- [ ] Confirmar campos e fontes para plano, pilar, diretoria, unidade, período e metas.
- [ ] Implementar consultas consolidadas no repository, sem SQL na view ou JavaScript.
- [ ] Implementar service para cálculos, denominadores nulos, arredondamento e regras temporais.
- [ ] Criar cards de indicadores e lançamentos conforme dados disponíveis.
- [ ] Criar resumos de homologações pendentes, aprovadas e rejeitadas.
- [ ] Implementar filtros por ano, mês, plano, pilar, diretoria, unidade e status quando suportados pelo esquema.
- [ ] Aplicar filtros de modo consistente a cards, gráficos e atualizações recentes.
- [ ] Restringir dados pelo perfil e escopo da unidade do usuário.
- [ ] Criar gráficos de barras, linhas e rosca com Chart.js local e acessível.
- [ ] Exibir estados de carregamento, vazio, indisponibilidade e erro sem vazar detalhes.
- [ ] Implementar endpoint de resumo e endpoint de gráficos com respostas padronizadas.
- [ ] Avaliar índices com planos de execução e justificar qualquer script SQL proposto.
- [ ] Criar testes dos cálculos, filtros combinados, escopo, períodos sem dados e desempenho.

## Critérios de aceite

- [ ] Métricas foram homologadas pela área de negócio com amostras reconciliadas no banco.
- [ ] Cards e gráficos respeitam os mesmos filtros e o mesmo escopo de acesso.
- [ ] O dashboard lida corretamente com ausência de dados e divisão por zero.
- [ ] O tempo de resposta atende ao limite definido para o volume real.
